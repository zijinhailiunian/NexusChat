# Definition of Done (DoD)

为确保每个任务都可以客观判断“是否完成”，NexusChat 采用以下统一 DoD（Definition of Done）标准。

## DoD 检查项（每个功能都必须满足）

- [ ] 1. 代码实现完成
- [ ] 2. 单元测试通过
- [ ] 3. 接口/行为文档更新
- [ ] 4. 日志与错误处理到位
- [ ] 5. 代码评审通过

## 使用方式

- 每个合并请求（PR）都必须逐项勾选以上 5 条。
- 若某项不适用（N/A），需在 PR 描述中给出明确理由。
- 合并前由评审者复核勾选结果与证据（测试输出、文档链接、关键代码位置）。

## 推荐证据

- 代码实现完成：关联提交与主要文件路径。
- 单元测试通过：`npm test` 或 `npm run check` 输出。
- 接口/行为文档更新：`docs/api.md` / `README.md` / 其他文档变更。
- 日志与错误处理到位：错误路径说明、日志字段示例、相关测试。
- 代码评审通过：至少 1 名 reviewer 批准（按仓库策略执行）。


## 测试策略执行要求
- 每次提交至少执行：`npm run lint` + `npm run test:unit`。
- 合并前至少执行：`npm run test:all`（或项目定义的核心回归集）。
- 推荐在 CI 执行：`npm run check`。
